Honest Validator Self-Equivocation
in TON Simplex Consensus
SkipVote + NotarizeVote broadcast for the same slot via two independent code paths, violating the single-vote invariant without Byzantine behaviour.
Repository ton-blockchain/ton
Branch testnet
Commit 3bb6abc
Scope validator/consensus/simplex/
Date 14 March 2026
⚠ Severity: Medium — Correctness violation with bounded liveness impact

Executive Summary

An honest Simplex validator can broadcast both a SkipVote and a NotarizeVote for the same consensus slot, producing a self-equivocation without any compromise or Byzantine behaviour. This violates the single-vote-per-slot invariant in the consensus implementation.

The bug manifests through two independent code paths sharing the same root cause — an incomplete boolean guard that checks !voted_final but omits !voted_notar and !voted_skip:

  • Path A — alarm() race: The alarm timer and try_notarize() coroutine use disjoint guards, allowing both to broadcast votes for the same slot concurrently.
  • Path B — start_up() restart: On every restart, a validator broadcasts SkipVote for its active window slots without checking whether it already cast NotarizeVote before the crash.

Both paths are dynamically confirmed via the test-consensus harness.

Impact Assessment

Path A — alarm() Race Condition

Root Cause

alarm() and try_notarize() share state on a slot but use disjoint boolean guards. alarm() checks only !voted_final before broadcasting SkipVote. try_notarize() sets voted_notar and broadcasts NotarizeVote after resuming from co_await ValidationRequest without checking whether a SkipVote was sent during the suspension.

validator/consensus/simplex/consensus.cpp

Bug A1 — alarm(): missing !voted_notar guard

// consensus.cpp ~line 144
td::actor::Task<> alarm() {
  for (size_t i = first_...; i < last_...; ++i) {
    auto slot = bus_->slots.get(i);
    if (slot && !slot->state->voted_final) {       // ← MUST also check !voted_notar && !voted_skip
      owning_bus().publish<BroadcastVote>(SkipVote{i});
    }
  }
}

Bug A2 — try_notarize(): missing !voted_skip check after co_await

// consensus.cpp ~line 228
td::actor::Task<> try_notarize(size_t slot_idx, ...) {
  auto res = co_await ValidationRequest{candidate};
  // ← alarm() may have fired during co_await and broadcast SkipVote
  slot->state->voted_notar = candidate->id;
  owning_bus().publish<BroadcastVote>(NotarizeVote{...});  // ← MUST check !voted_skip first
}

Bug A3 — check_invariants(): missing detection case

// pool.cpp ~line 209
void check_invariants() {
  if (finalize_.has_value() && skip_.has_value()) { ... report ... }
  if (finalize_.has_value() && notarize_.has_value()) { ... report ... }
  // MISSING: notarize_ + skip_ combination not detected:
  // if (notarize_.has_value() && skip_.has_value()) { report misbehavior; }
}

Path B — start_up() Restart Equivocation

Root Cause

On restart, start_up() loads bootstrap_votes from the persistent DB (including any NotarizeVote cast before the crash), then immediately broadcasts SkipVote for all slots in the current first_nonannounced_window range using only a !voted_final guard. Peers who received the pre-crash NotarizeVote now also receive a SkipVote from the same validator for the same slot.

validator/consensus/simplex/consensus.cpp
// consensus.cpp ~line 82
void start_up() {
  ...
  if (auto window = bus->first_nonannounced_window) {
    for (size_t i = *window; i < last_slot; ++i) {
      auto slot = bus->slots.get(i);
      if (slot.has_value() && !slot->state->voted_final) {   // ← MUST also check !voted_notar
        owning_bus().publish<BroadcastVote>(SkipVote{i});
      }
    }
  }
}

Database persistence: The test-consensus harness uses TestDbImpl backed by shared_ptr<DbInner> (test-consensus.cpp:610, 689). The DB is preserved across gremlin kills, so bootstrap_votes correctly reflects pre-crash state on restart — matching production behaviour.
